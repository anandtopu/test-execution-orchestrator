# TEO v1.0.0

**Test Execution Orchestrator — first stable release.**

TEO is a CI-agnostic control plane for parallel test execution, result aggregation, and flaky-test detection. It targets teams whose CI wall-clock has outgrown a single runner. The product wedges are prediction-driven scheduling (LPT bin-packing on predicted durations), trace-native results (OpenTelemetry over JUnit XML), and statistical flake quarantine (Wilson interval, bounded false-positive rate).

This is the first stable release. Everything below is included in the box.

## What's in the box

**Core platform**
- Run intake via REST (`POST /api/v1/runs`), idempotent on `Idempotency-Key`, with per-run budget enforcement
- Run Manager state machine (8 states, Postgres advisory-lock leader election) — replayable scheduler plans, budget enforcement, observer pattern for downstream integrations
- LPT scheduler with property test verifying makespan ≤ 4/3 × OPT, exclusivity tags, quarantine lane, deterministic tie-break
- Worker agent (Postgres SKIP-LOCKED claim path, NATS push, IMDSv2 spot-interruption draining), with adapters for **pytest**, **go test**, and **jest**
- OTLP gRPC receiver writing to ClickHouse via native `PrepareBatch` (column-major, one round-trip per Export); ERROR-status spans routed to failure-cluster fingerprinting
- Wilson-interval flake detection over ≥20 runs with auto-quarantine into a non-blocking lane and broken-vs-flaky distinction
- Auto-quarantine workflow with GitHub Issue creation, CODEOWNERS-resolved assignees, SLA nudges, un-quarantine magic-link restore
- Weekly per-author owner digest (SMTP + Slack) with WoW delta and per-user / per-repo opt-out
- Python LightGBM predictor service with champion/challenger gating against the Go heuristic baseline

**Surfaces**
- REST API with chi router, JWT (HS256) issuer + argon2id-hashed API keys (30s revocation cache), audit log, RFC 7807 errors
- gRPC Workers service on `:9090` — **binary protobuf wire format** via the generated `internal/proto/teov1` bindings (the v1.0 release retires the JSON-codec stand-in)
- GraphQL read API (`/graphql`) covering runs, run-by-id, shards, failure clusters, flakes, weekly cost summary, and the `rerunFailed` mutation
- Web UI (Next.js 15) with run list, run detail (Gantt + 2s polling, auto-stops on terminal), failure clusters, flakes, **cost dashboard**, and rerun-failed
- **Run-export endpoints** (`GET /api/v1/runs/{id}/export?format={junit|otlp}`) — JUnit XML for legacy CI imports, OTLP protobuf (or `?as=json`) for tracing backends
- GitHub App integration: webhook HMAC verification, Check Runs with top-3 failure-cluster summaries and deep links

**Operations**
- Helm umbrella chart with API / Run Manager / result-pipeline Deployments, pre-upgrade migration Job, three CronJobs (quarantine, owner digest, ML train), Karpenter NodePools
- Five Grafana dashboards + six PrometheusRule alerts (with `runbook_url` annotations) shipped as ConfigMaps with sidecar discovery labels
- Subcharts vendored & toggleable: CloudNativePG, ClickHouse Operator, NATS, MinIO, Dex
- `teo doctor` CLI with parallel dependency checks (`--json` for scripting)
- `/healthz`, `/readyz`, `/metrics` on every service — the bundled dashboards and PrometheusRules reference the exact metric names emitted by `internal/metrics`
- **Per-test stdout/stderr** captured, redacted, and uploaded to S3 via the AWS SDK transfermanager (auto-multipart >16MB); object key persisted into `test_executions.log_object_key`
- **Cost dashboard** at `/cost` shows weekly `$/build` trend and spot-vs-on-demand share; pricing configurable via `TEO_COST_SPOT_PER_MIN` / `TEO_COST_ONDEMAND_PER_MIN`
- Restore drill runbook at `docs/operations/restore-drill.md` (8-section reproducible procedure)

**Quality bar**
- 20 unit packages green; 38 integration tests under `-tags=integration` (Postgres + MinIO testcontainers); 6 frontend Vitest files
- License compliance: Apache 2.0 throughout; CI fails on any AGPL/forbidden transitive dep
- Trivy HIGH/CRITICAL gate on every container image
- Cosign keyless signing for archives + checksums; CycloneDX SBOMs per artifact (Syft)

## Architecture quick-reference

| Component | Stack | Where it lives |
|---|---|---|
| Backend services + CLI | Go 1.25 | `cmd/{api,run-manager,scheduler,result-pipeline,worker,predictor,teo}` |
| State / time-series / blobs | Postgres + ClickHouse + S3-compatible | `migrations/postgres`, `migrations/clickhouse` |
| Streaming | NATS JetStream | `internal/nats/` |
| ML predictor | Python + LightGBM (FastAPI, S3-backed model registry) | `services/predictor-ml/` |
| Web UI | Next.js 15 + GraphQL via urql | `web/` |
| Deploy | Helm umbrella chart + Karpenter (AWS) | `deploy/helm/teo/` |

20 ADRs in [`docs/adr/`](../adr/) document every material decision (storage, OTel-as-source-of-truth, Wilson interval, advisory-lock leader election, MVP scope cut, etc.).

## Installation

```bash
# Add the chart repo
helm repo add teo https://teo-dev.github.io/teo
helm repo update

# Install with sensible defaults (bundles CloudNativePG, ClickHouse Operator, NATS, MinIO, Dex)
helm install teo teo/teo \
  --create-namespace --namespace teo \
  --set auth.jwt.secretName=<your-secret> \
  --set auth.jwt.secretKey=jwt-secret
```

To bring your own managed services (RDS Postgres, ClickHouse Cloud, etc.), disable the relevant subchart with `--set <name>.enabled=false` and provide the DSN via `TEO_*` env vars. The chart-testing CI job templates this on/off matrix on every PR.

The `teo` CLI binaries ship as signed `tar.gz` (linux/darwin amd64/arm64) and `zip` (windows amd64/arm64) archives on the GitHub Releases page, with cosign signatures + CycloneDX SBOMs alongside.

## Compatibility

- **Kubernetes**: tested on kind 1.29 + 1.30; AWS-only for v1 (Karpenter NodePools, IMDSv2 spot-interruption polling)
- **Postgres**: 16+
- **ClickHouse**: 24+
- **Go (build-from-source)**: 1.25+

## Notes for operators

- **gRPC clients**: the v1.0 Workers service uses standard binary protobuf — the `:9090` listener is now reachable from any vanilla gRPC client without a custom JSON codec.
- **AWS pricing rates** for the cost dashboard default to m5.xlarge ballpark numbers ($0.012/min spot, $0.040/min on-demand). Operators with cost-explorer data should override via `TEO_COST_SPOT_PER_MIN` and `TEO_COST_ONDEMAND_PER_MIN`.
- **Per-test log uploads** require `TEO_S3_BUCKET` to be set on the worker; otherwise the worker uses an in-memory noop and `log_object_key` stays NULL on `test_executions`.
- **Restore drill**: rehearse the 8-section procedure in `docs/operations/restore-drill.md` against staging before relying on the backup posture in production. The runbook includes a history log at the bottom; record drill outcomes there.
